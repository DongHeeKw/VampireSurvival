# .gitmessage.txt
# [#이슈번호] <타입>: <제목>

# ==== 제목 작성 가이드 ====
# - 제목은 50자 이내로 작성
# - 마침표 없이 작성
# - 과거형을 사용하지 않고 명령조로 작성
# 예) "Fixed" -> "Fix", "Added" -> "Add"
# ========================

# ==== 본문 작성 가이드 ====
# - What - 무엇을 변경했는지
# - Why - 왜 변경했는지
# - How - 어떻게 변경했는지
# ========================

# ==== 꼬리말 작성 가이드 ====
# - 이슈 트래커 ID 작성
# - Fixes: (이슈 수정시 사용)
# - Refs: (참고 이슈)
# ========================

# ==== 커밋 타입 ====
# feat: 새로운 기능 추가
# fix: 버그 수정
# docs: 문서 수정
# style: 코드 포맷팅, 세미콜론 누락, 코드 변경이 없는 경우
# refactor: 코드 리팩토링
# test: 테스트 코드 추가
# chore: 빌드 업무 수정, 패키지 매니저 수정
# design: CSS 등 사용자 UI 디자인 변경
# comment: 필요한 주석 추가 및 변경
# rename: 파일 혹은 폴더명을 수정만 한 경우
# remove: 파일을 삭제만 한 경우
# !BREAKING CHANGE: 커다란 API 변경의 경우
# !HOTFIX: 급하게 치명적인 버그를 고쳐야 하는 경우
# ========================

# ==== 예시 ====
# [#123] feat: Add cube evolution system
#
# - 큐브 진화 시스템 기본 로직 구현
# - 속성별 진화 경로 추가
# - 진화 시 파티클 효과 구현
#
# Fixes: #123
# Refs: #456
# ========================
